📝 INSTRUCCIONES PARA IMPORTACIÓN MASIVA - SGIR

Para realizar una carga masiva exitosa en el Sistema de Gestión de Infraestructura y Respaldos, siga estas directrices:

1. FORMATO DEL ARCHIVO:
   - El archivo debe ser un CSV (Valores separados por comas).
   - Mantenga la cabecera (primera fila) exactamente como está en la plantilla.

2. COLUMNAS Y VALORES PERMITIDOS:
   - nombre_servidor: Nombre descriptivo (ej. Srv-DB-Prod).
   - direccion_ip: IPv4 o IPv6 válida.
   - es_legacy: 'true' para conexiones SSH antiguas, 'false' para modernas.
   - nivel_criticidad: Bajo, Medio, Alto, Crítico.
   - estado: Activo, Inactivo.
   - nombre_dbms: Oracle Database, MySQL, PostgreSQL, MongoDB (o dejar vacío).
   - nombre_instancia: SID o Service Name (ej. ORCL).
   - tipo_acceso: SSH, DB Native.

3. CONSOLIDACIÓN POR IP:
   - Si desea agregar múltiples instancias o credenciales a un mismo servidor, repita la fila del servidor con la misma IP pero cambie los datos de la instancia/credencial. El sistema creará un solo servidor y le asignará todos los elementos relacionados.

4. SEGURIDAD:
   - Las contraseñas se escriben en texto plano en el CSV. 
   - El sistema las cifrará automáticamente con AES-256 al procesarlas.
   - SE RECOMIENDA ELIMINAR EL ARCHIVO CSV DE SU EQUIPO UNA VEZ COMPLETADA LA CARGA.
