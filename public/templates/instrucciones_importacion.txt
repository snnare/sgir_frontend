INSTRUCCIONES PARA CARGA MASIVA (CSV) - SGIR
============================================

1. FORMATO DE ARCHIVO:
   - El archivo debe ser extensión .csv
   - Codificación sugerida: UTF-8.
   - El separador debe ser coma ( , ).

2. ESTRUCTURA LÓGICA:
   - El motor lee la primera fila como cabecera (no modifique los nombres).
   - El sistema usa la 'direccion_ip' como llave principal para no duplicar servidores en múltiples filas.

3. COLUMNAS:
   - nombre_servidor (Obligatorio): El hostname o alias del servidor (ej. SRV-PROD-01).
   - direccion_ip (Obligatorio): Dirección IPv4 o IPv6. Llave principal.
   - es_legacy (Opcional): Define si el host requiere algoritmos SSH antiguos. Valores: true / false (Por defecto: false).
   - nivel_criticidad (Obligatorio): Valores admitidos: Bajo, Medio, Alto, Crítico.
   - descripcion (Opcional): Texto libre para notas o detalles adicionales.
   - estado (Opcional): Estado lógico del servidor. Valores admitidos: Activo, Inactivo.
   - nombre_dbms (Opcional): Motor de BD. Valores: Oracle Database, MySQL, PostgreSQL, MongoDB.
   - nombre_instancia (Opcional): Nombre de la base de datos o SID.
   - puerto_db (Opcional): Puerto de escucha (ej. 1521, 3306).
   - usuario (Opcional): Nombre de usuario de la credencial.
   - password (Opcional): Contraseña en texto plano. Se cifrará en destino.
   - tipo_acceso (Opcional): Valores admitidos: SSH, DB Native.
   - particiones (Opcional): Rutas de montaje. (/, /u01, /data) o separadas por comas. / por defecto.

4. NOTAS IMPORTANTES:
   - No cambies el nombre de los encabezados (primera fila).
   - Si una fila falla, se informará el error y fila, pero el resto se procesará.
